How it works

You set up once. The proxy handles every paid request from then on.

1. Connect wallet → SIWE challenge → tenant created
2. Add routes → method + path + price + your backend URL
3. We give you:  https://acme.kite.dev

   GET /forecast      ($0.05 USDC)   →  https://api.acme.io/forecast
   POST /v1/embed     ($0.01 USDC)   →  https://api.acme.io/v1/embed

4. A customer hits your proxy URL.

   • No payment header  →  HTTP 402 with price + payTo
   • Valid X-PAYMENT    →  forwarded upstream, response returned, USDC settled
                            (99% to you, 1% to us)
   • Every request logged → /__x402/metrics, /__x402/events

Live demo (right now)

The backend is up. Try it from any terminal: 

# Health probe — no payment required
curl https://x402-saas.onrender.com/__x402/health

# SIWE challenge — returns a nonce-signed message you'd sign with your wallet
curl -X POST https://x402-saas.onrender.com/api/v1/auth/challenge \
  -H "content-type: application/json" \
  -d '{"walletAddress":"0xC504Fd656330A823C3ffcBAB048c05cF45F60Bdf"}'

Backend runs on Render (durable host, free tier — first request after idle takes ~30s). Real x402.rs facilitator wired in — verify and settle live against Base. 38/38 tests passing on the codebase.
